# User Interface - Demo

The last step is to prepare a user interface to demo our work.
The UI should be composed of two panels - left and right.
The left panel should show all the plants on a map as a scatter plot, sized and colored like in @TASKS/04_exploring_bm_unit_matches.md.
Above the map should be a multi-select bar where the user can filter the map for dukes plants.
The right panel should show some matching details.
Put the @ui/im.png at the top of the right panel as a logo (you may move it from its current location in the repo).
At the top should be the active date, with a left and right arrow to change the active date.
Below this should be a multi-select bar where the user can filter for bm units.
When the user selects a bm unit, it populates a line plot showing the generation of that bm unit, and the generation of all dukes plants matching that bm unit.
The residual value (i.e. the difference) should be calculated and also shown on the line plot.
The way this should work, is whether the user selects a plant or a bm unit, it is the _group_ that is rendered on both panels.

The UI should load groups.json from the public google bucket, but it can be seen here at @data/groups.json.
The daily actual and estimated generation will also be available to be read from the public google bucket.
The application should be a single-page react app with its root at @ui/.
Deployment will be with cloudflare.
Keep the styling tidy, minimal, and techy.

Plan first and ask any clarification necessary.
Update clarifications on this document.
When your're done executing, update the @CLAUDE.md with anything you've learned.

## Implementation notes

- `scripts/prepare_plants_json.py` generates `data/plants.json` (DUKES plants with group_id) — run before deploying
- `VITE_DATA_BASE_URL` env var sets GCS base URL; empty = local `/data/` (symlink in `ui/public/data` → `../../data`)
- `plant_generation` in inference only covers wind + solar; the full DUKES map uses all 8 technology types
- Line chart sums all BM unit actuals and all matched plant estimates per group; residual = actual − estimated
- Group selection works bidirectionally: map click → group → chart; BM unit search → group → map highlight
- Deployment: `cd ui && npm run build` → `dist/` → Cloudflare Pages
